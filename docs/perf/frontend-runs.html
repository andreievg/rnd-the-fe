Frontend perf bake-off — results

Comparable frontend performance numbers for the team bake-off. Every row is produced by the same versioned collector (collect.js) under identical CPU/network throttle. The highlighted green columns are the headline, content-based metrics — TTNetworkQuiet (primary ranking metric) and TTDataRendered. All timings are the median of N runs with the warm-up run dropped; the (min–max) in parentheses is the spread.

Read me before comparing:
  • Do not rank on the load event or LCP. For a client-rendered SPA the load event fires long before the data the user came for has arrived (we measured load at ~441ms vs data settling at ~4.2s on a real app). Rank on TTNetworkQuiet / TTDataRendered.
  • † LCP is often blank for SPAs — that is expected, not an error. The largest content (e.g. a table) is client-rendered after first interaction, so no LCP candidate is recorded.
  • A row is only comparable to another with a similar data-item count and the same throttle. A 50-row page and a 2000-row page are not comparable.
